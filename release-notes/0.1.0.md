# Aqua v0.1.0 - Thermals & Units

## Thermals - Complete UI Overhaul

**Per-sensor popovers** - Clicking anywhere on the Thermals card used to open one big generic menu. Each sensor chip (CPU, Memory, Ambient, etc.) is now individually tappable and opens its own focused popover showing a history sparkline, min/avg/max stats, the raw sensor source name, and warm/hot thresholds.

**Per-sensor history** - CPU, GPU, Memory, and Ambient temperatures are now tracked in separate rolling history buffers. Each chip's popover shows history specific to that sensor, not just the hottest reading overall.

**"All sensors" inline dropdown** - A new disclosure row below the chip grid expands the Thermals card in-place to show every individual sensor reading the hardware exposes. Readings are grouped by category (CPU / GPU / Memory / Ambient), each with its source sensor name and a plain-English description of what it measures.

**CPU die grid** - When multiple CPU thermal probes are present (Apple Silicon exposes one per die region), they're displayed as a compact 4-column grid sorted numerically - Die 1 through Die N - rather than a confusing list of identical "CPU" rows.

**Temperature color gradient** - Sensor chips now use a smooth blue -> teal -> yellow -> orange -> red gradient instead of a binary warm/danger color jump. Colors are scaled relative to each sensor's own danger threshold, so CPU, Memory, and Ambient all feel appropriately calibrated.

**Ambient threshold fix** - The Ambient sensor on Apple Silicon reads from an internal board calibration probe (`tcal`), not outside air. Warning and danger thresholds have been corrected from 35/45°C to 60/75°C - 52°C is completely normal and will no longer show as red.

**Popover stability fix** - Thermal chip popovers previously dismissed on their own during polling ticks because each update regenerated new view IDs, resetting SwiftUI state. Chips now use stable string IDs so open popovers persist correctly across live data updates.

## CPU Cores Card

**Die temperature in expanded view** - Double-clicking the CPU Cores card to expand it now shows the live CPU die temperature in the summary row alongside core average and busiest core. The temperature uses the same gradient color as the Thermals card and respects the F/C unit preference.

## Settings - Units

**Fahrenheit / Celsius toggle** - A new **Units** section in Settings lets you switch all temperature displays app-wide between °C and °F. The preference is applied everywhere: the toolbar badge, Thermals chip faces, chip popovers (including history chart axis and stats), the All Sensors dropdown, the CPU Cores expanded summary, and the CPU detail popover.
